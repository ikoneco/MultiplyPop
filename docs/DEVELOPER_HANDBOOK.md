# Developer & AI Handbook

## Introduction
This handbook outlines the workflows, conventions, and rules for contributing to MultiplyPop. It is designed for both human developers and AI assistants.

## Workflow Overview
1. **Start Session**: Pull latest code, read `PLAN.md`, `TASKS.md`, etc.
2. **Execute**: Implement changes, following `ARCH_NOTES.md`.
3. **Document**: Update `SUMMARY.md`, `CHANGELOG.md`, etc.
4. **End Session**: Push to GitHub.

## Code Style
- **TypeScript**: Strict mode enabled.
- **Styling**: Tamagui for all UI components.
- **State**: Zustand for global state.

## Git Conventions
- **Commits**: Use descriptive messages (e.g., "feat: add user profile", "fix: login error").
- **Branches**: `feature/name-of-feature` for new work.

## AI Handoff
Refer to `PLAN.md` for the strict AI handoff protocol.
